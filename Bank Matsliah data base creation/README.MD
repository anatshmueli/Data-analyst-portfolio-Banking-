# 🏦 Bank Matsliah – Database Design & SQL

## 📌 Project Overview

This project focuses on the design and implementation of a relational database for **Bank Matsliah** using Microsoft SQL Server.

The database was designed following database normalization principles and includes relationships, primary keys, foreign keys, and constraints to ensure data integrity and support future analytical projects.

This database serves as the foundation for future data analysis using SQL, Python, and Power BI.

---

## 🎯 Project Objectives

- Design a relational database from scratch.
- Build a scalable banking database.
- Define Primary Keys and Foreign Keys.
- Maintain data integrity using Constraints.
- Prepare the database for business analysis and reporting.

---

## 🏦 Business Scenario

The project is based on a banking environment.

The database stores information about:

- 👨‍💼 Bank employees
- 👥 Clients
- 💳 Bank accounts
- 💰 Deposits
- 🏠 Loans & Mortgages

The data model reflects real-world banking relationships between customers, accounts, employees, deposits, and loans.

---

## 🗂 Database Structure

The database contains **5 relational tables**:

| Table | Description |
|--------|-------------|
| **Bankers** | Bank employees and reporting hierarchy |
| **Clients** | Customer information |
| **Accounts** | Bank accounts and balances |
| **Deposits** | Customer deposits |
| **Loans** | Loans and mortgages |

---

## 🔗 Database Relationships

- Bankers → Accounts
- Clients → Accounts
- Accounts → Deposits
- Accounts → Loans
- Bankers → Bankers (Manager hierarchy)

The database uses:

- Primary Keys
- Foreign Keys
- Constraints
- One-to-Many Relationships
